Extension(String)

Returns the extension without the leading period character (.) if the string is a valid file name or path. If the string does not contain an extension, returns an empty string instead.

Declaration
public static string Extension(this string instance)
Parameters
Type Name Description
System.String instance

The path to a file.
Returns
Type Description
System.String

The extension of the file or an empty string.
Examples
GD.Print("/path/to/file.txt".GetExtension())  // "txt"
GD.Print("file.txt".GetExtension())  // "txt"
GD.Print("file.sample.txt".GetExtension())  // "txt"
GD.Print(".txt".GetExtension())  // "txt"
GD.Print("file.txt.".GetExtension())  // "" (empty string)
GD.Print("file.txt..".GetExtension())  // "" (empty string)
GD.Print("txt".GetExtension())  // "" (empty string)
GD.Print("".GetExtension())  // "" (empty string)

GetStringFromASCII(Byte[])

Converts the given byte array of ASCII encoded text to a string. Faster alternative to GetStringFromUTF8(Byte[]) if the content is ASCII-only. Unlike the UTF-8 function this function maps every byte to a character in the array. Multibyte sequences will not be interpreted correctly. For parsing user input always use GetStringFromUTF8(Byte[]).

Declaration
public static string GetStringFromASCII(this byte[] bytes)
Parameters
Type Name Description
System.Byte[] bytes

A byte array of ASCII characters (on the range of 0-127).
Returns
Type Description
System.String

A string created from the bytes.

Converts the given byte array of UTF-8 encoded text to a string. Slower than GetStringFromASCII(Byte[]) but supports UTF-8 encoded data. Use this function if you are unsure about the source of the data. For user input this function should always be preferred.

Converts the String (which is a character array) to PoolByteArray (which is an array of bytes). The conversion is speeded up in comparison to ToUTF8(String) with the assumption that all the characters the String contains are only ASCII characters.

ToUTF8(String)

Converts the String (which is an array of characters) to PoolByteArray (which is an array of bytes). The conversion is a bit slower than ToAscii(String), but supports all UTF-8 characters. Therefore, you should prefer this function over ToAscii(String).
